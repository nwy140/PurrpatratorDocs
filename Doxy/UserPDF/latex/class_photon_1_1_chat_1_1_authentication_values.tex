\doxysubsection{Detailed Description}
Container for user authentication in \doxyref{Photon}{p.}{namespace_photon}. 

Set Auth\+Values before you connect -\/ all else is handled. 

On \doxyref{Photon}{p.}{namespace_photon}, user authentication is optional but can be useful in many cases. If you want to Find\+Friends, a unique ID per user is very practical.

There are basically three options for user authentication\+: None at all, the client sets some User\+Id or you can use some account web-\/service to authenticate a user (and set the User\+Id server-\/side).

Custom Authentication lets you verify end-\/users by some kind of login or token. It sends those values to \doxyref{Photon}{p.}{namespace_photon} which will verify them before granting access or disconnecting the client.

The Auth\+Values are sent in Op\+Authenticate when you connect, so they must be set before you connect. If the Auth\+Values.\+User\+Id is null or empty when it\textquotesingle{}s sent to the server, then the \doxyref{Photon}{p.}{namespace_photon} Server assigns a User\+Id!

The \doxyref{Photon}{p.}{namespace_photon} Cloud Dashboard will let you enable this feature and set important server values for it. {\texttt{ https\+://dashboard.\+photonengine.\+com}}
