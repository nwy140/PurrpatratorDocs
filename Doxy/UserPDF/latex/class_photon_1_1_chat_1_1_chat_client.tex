\doxysubsection{Detailed Description}
Central class of the \doxyref{Photon}{p.}{namespace_photon} \doxyref{Chat}{p.}{namespace_photon_1_1_chat} API to connect, handle channels and messages. 

This class must be instantiated with a \doxyref{IChat\+Client\+Listener}{p.}{interface_photon_1_1_chat_1_1_i_chat_client_listener} instance to get the callbacks. Integrate it into your game loop by calling Service regularly. If the target platform supports Threads/\+Tasks, set Use\+Background\+Worker\+For\+Sending = true, to let the \doxyref{Chat\+Client}{p.}{class_photon_1_1_chat_1_1_chat_client} keep the connection by sending from an independent thread.

Call Connect with an App\+Id that is setup as \doxyref{Photon}{p.}{namespace_photon} \doxyref{Chat}{p.}{namespace_photon_1_1_chat} application. Note\+: Connect covers multiple messages between this client and the servers. A short workflow will connect you to a chat server.

Each \doxyref{Chat\+Client}{p.}{class_photon_1_1_chat_1_1_chat_client} resembles a user in chat (set in Connect). Each user automatically subscribes a channel for incoming private messages and can message any other user privately. Before you publish messages in any non-\/private channel, that channel must be subscribed.

Public\+Channels is a list of subscribed channels, containing messages and senders. Private\+Channels contains all incoming and sent private messages.

{\footnotesize\ttfamily bool Photon.\+Chat.\+Chat\+Client.\+Add\+Friends (\begin{DoxyParamCaption}\item[{string[$\,$]}]{friends }\end{DoxyParamCaption})}



Adds friends to a list on the \doxyref{Chat}{p.}{namespace_photon_1_1_chat} Server which will send you status updates for those. 

Add\+Friends and Remove\+Friends enable clients to handle their friend list in the \doxyref{Photon}{p.}{namespace_photon} \doxyref{Chat}{p.}{namespace_photon_1_1_chat} server. Having users on your friends list gives you access to their current online status (and whatever info your client sets in it).

Each user can set an online status consisting of an integer and an arbitrary (serializable) object. The object can be null, Hashtable, object[] or anything else \doxyref{Photon}{p.}{namespace_photon} can serialize.

The status is published automatically to friends (anyone who set your user ID with Add\+Friends).

\doxyref{Photon}{p.}{namespace_photon} flushes friends-\/list when a chat client disconnects, so it has to be set each time. If your community API gives you access to online status already, you could filter and set online friends in Add\+Friends.

Actual friend relations are not persistent and have to be stored outside of \doxyref{Photon}{p.}{namespace_photon}.

The client will remove these channels from the Public\+Channels dictionary once the server sent a response to this request.

The request will be sent to the server and \doxyref{IChat\+Client\+Listener.\+On\+Unsubscribed}{p.}{interface_photon_1_1_chat_1_1_i_chat_client_listener_a54c546b010d3578316532d0af31ee183} gets called when the server actually removed the channel subscriptions.

Unsubscribe will fail if you include null or empty channel names.

Defines if a background thread will call Send\+Outgoing\+Commands, while your code calls Service to dispatch received messages. 

The benefit of using a background thread to call Send\+Outgoing\+Commands is this\+:

Even if your game logic is being paused, the background thread will keep the connection to the server up. On a lower level, acknowledgements and pings will prevent a server-\/side timeout while (e.\+g.) Unity loads assets.

Your game logic still has to call Service regularly, or else incoming messages are not dispatched. As this typically triggers UI updates, it\textquotesingle{}s easier to call Service from the main/\+UI thread.
