Depending on the \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view}\textquotesingle{}s Ownership\+Transfer setting, any client can request to become owner of the \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view}. 

Requesting ownership can give you control over a \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view}, if the Ownership\+Transfer setting allows that. The current owner might have to implement IPun\+Callbacks.\+On\+Ownership\+Request to react to the ownership request.

The owner/controller of a \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view} is also the client which sends position updates of the Game\+Object.

\doxysubsubsection{Owner}
{\footnotesize\ttfamily \textbf{ Player} Photon.\+Pun.\+Photon\+View.\+Owner\hspace{0.3cm}{\ttfamily [get]}}



The owner of a \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view} is the creator of an object by default Ownership can be transferred and the owner may not be in the room anymore. 

Objects in the scene don\textquotesingle{}t have an owner. 

The owner/controller of a \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view} is also the client which sends position updates of the Game\+Object.

Ownership can be transferred to another player with \doxyref{Photon\+View.\+Transfer\+Ownership}{p.}{class_photon_1_1_pun_1_1_photon_view_a634fee6dff393659ff6578680792b230} or any player can request ownership by calling the \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view}\textquotesingle{}s Request\+Ownership method. The current owner has to implement IPun\+Callbacks.\+On\+Ownership\+Request to react to the ownership request.
