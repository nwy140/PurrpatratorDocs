\doxysubsection{Detailed Description}
Class wrapping the received \doxyref{Event\+Code.\+Error\+Info}{p.}{class_photon_1_1_realtime_1_1_event_code_a8ac7b996b1b15eb56a0e8eb3342cea8c} event. 

This is passed inside \doxyref{IError\+Info\+Callback.\+On\+Error\+Info}{p.}{interface_photon_1_1_realtime_1_1_i_error_info_callback_a383d0328f4cc063c87d89213a985845a} callback. If you implement \doxyref{IOn\+Event\+Callback.\+On\+Event}{p.}{interface_photon_1_1_realtime_1_1_i_on_event_callback_a0e69d8b1154c5438d5e5a224dd869e45} or \doxyref{Load\+Balancing\+Client.\+Event\+Received}{p.}{class_photon_1_1_realtime_1_1_load_balancing_client_aa8680c4d6c86d282b8b2b51219aeaaf0} you will also get \doxyref{Event\+Code.\+Error\+Info}{p.}{class_photon_1_1_realtime_1_1_event_code_a8ac7b996b1b15eb56a0e8eb3342cea8c} but not parsed.

In most cases this could be either\+:
\begin{DoxyEnumerate}
\item an error from webhooks plugin (if Has\+Error\+Info is enabled), read more here\+: {\texttt{ https\+://doc.\+photonengine.\+com/en-\/us/realtime/current/gameplay/web-\/extensions/webhooks\#options}}
\item an error sent from a custom server plugin via Plugin\+Host.\+Broadcast\+Error\+Info\+Event, see example here\+: {\texttt{ https\+://doc.\+photonengine.\+com/en-\/us/server/current/plugins/manual\#handling\+\_\+http\+\_\+response}}
\item an error sent from the server, for example, when the limit of cached events has been exceeded in the room (all clients will be disconnected and the room will be closed in this case) read more here\+: {\texttt{ https\+://doc.\+photonengine.\+com/en-\/us/realtime/current/gameplay/cached-\/events\#special\+\_\+considerations}} 
\end{DoxyEnumerate}
