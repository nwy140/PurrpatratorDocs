\doxysubsection{Detailed Description}
Defines an interface for object pooling, used in \doxyref{Photon\+Network.\+Instantiate}{p.}{class_photon_1_1_pun_1_1_photon_network_a3c3998fe3bc09a7de8d09378d7a3e2d0} and \doxyref{Photon\+Network.\+Destroy}{p.}{class_photon_1_1_pun_1_1_photon_network_ada7d7b9bdb306863c66b260dde7c544e}. 

To apply your custom \doxyref{IPun\+Prefab\+Pool}{p.}{interface_photon_1_1_pun_1_1_i_pun_prefab_pool}, set \doxyref{Photon\+Network.\+Prefab\+Pool}{p.}{class_photon_1_1_pun_1_1_photon_network_ab4dd61936d6f0060d7317607076071bf}.

The pool has to return a valid, disabled Game\+Object when PUN calls Instantiate. Also, the position and rotation must be applied.

Note that Awake and Start are only called once by Unity, so scripts on re-\/used Game\+Objects should make use of On\+Enable and or On\+Disable. When On\+Enable gets called, the \doxyref{Photon\+View}{p.}{class_photon_1_1_pun_1_1_photon_view} is already updated to the new values.

To be able to enable a Game\+Object, Instantiate must return an inactive object.

Before PUN \char`\"{}destroys\char`\"{} Game\+Objects, it will disable them.

If a component implements \doxyref{IPun\+Instantiate\+Magic\+Callback}{p.}{interface_photon_1_1_pun_1_1_i_pun_instantiate_magic_callback}, PUN will call On\+Photon\+Instantiate when the networked object gets instantiated. If no components implement this on a prefab, PUN will optimize the instantiation and no longer looks up \doxyref{IPun\+Instantiate\+Magic\+Callback}{p.}{interface_photon_1_1_pun_1_1_i_pun_instantiate_magic_callback} via Get\+Components.
