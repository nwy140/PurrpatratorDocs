\doxysubsection{Detailed Description}
Collection of \char`\"{}organizational\char`\"{} callbacks for the \doxyref{Realtime}{p.}{namespace_photon_1_1_realtime} Api to cover\+: Connection and Regions. 

Classes that implement this interface must be registered to get callbacks for various situations.

To register for callbacks, call \doxyref{Load\+Balancing\+Client.\+Add\+Callback\+Target}{p.}{class_photon_1_1_realtime_1_1_load_balancing_client_a7eccc149073296a316965f2ce96d08d3} and pass the class implementing this interface To stop getting callbacks, call \doxyref{Load\+Balancing\+Client.\+Remove\+Callback\+Target}{p.}{class_photon_1_1_realtime_1_1_load_balancing_client_a2b479687e6c8f0d73661447c91e41a88} and pass the class implementing this interface

\doxysubsubsection{OnConnected()}
{\footnotesize\ttfamily void Photon.\+Realtime.\+IConnection\+Callbacks.\+On\+Connected (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}



Called to signal that the \char`\"{}low level connection\char`\"{} got established but before the client can call operation on the server. 

After the (low level transport) connection is established, the client will automatically send the Authentication operation, which needs to get a response before the client can call other operations.

Your logic should wait for either\+: On\+Region\+List\+Received or On\+Connected\+To\+Master.

This callback is useful to detect if the server can be reached at all (technically). Most often, it\textquotesingle{}s enough to implement \doxyref{On\+Disconnected(\+Disconnect\+Cause cause)}{p.}{interface_photon_1_1_realtime_1_1_i_connection_callbacks_a5e133a672790851ab0554fdc821c7727} and check for the cause.

This is not called for transitions from the masterserver to game servers.

\doxysubsubsection{OnCustomAuthenticationFailed()}
{\footnotesize\ttfamily void Photon.\+Realtime.\+IConnection\+Callbacks.\+On\+Custom\+Authentication\+Failed (\begin{DoxyParamCaption}\item[{string}]{debug\+Message }\end{DoxyParamCaption})}



Called when the custom authentication failed. 

Followed by disconnect! 

Custom Authentication can fail due to user-\/input, bad tokens/secrets. If authentication is successful, this method is not called. Implement On\+Joined\+Lobby() or \doxyref{On\+Connected\+To\+Master()}{p.}{interface_photon_1_1_realtime_1_1_i_connection_callbacks_a52dfd77079d550168c7ff7aa0ce01a6c} (as usual).

During development of a game, it might also fail due to wrong configuration on the server side. In those cases, logging the debug\+Message is very important.

Unless you setup a custom authentication service for your app (in the {\texttt{ Dashboard}}), this won\textquotesingle{}t be called!

\doxysubsubsection{OnCustomAuthenticationResponse()}
{\footnotesize\ttfamily void Photon.\+Realtime.\+IConnection\+Callbacks.\+On\+Custom\+Authentication\+Response (\begin{DoxyParamCaption}\item[{Dictionary$<$ string, object $>$}]{data }\end{DoxyParamCaption})}



Called when your Custom Authentication service responds with additional data. 

Custom Authentication services can include some custom data in their response. When present, that data is made available in this callback as Dictionary. While the keys of your data have to be strings, the values can be either string or a number (in Json). You need to make extra sure, that the value type is the one you expect. Numbers become (currently) int64.

Example\+: void On\+Custom\+Authentication\+Response(Dictionary\texorpdfstring{$<$}{<}string, object\texorpdfstring{$>$}{>} data) \{ ... \}
