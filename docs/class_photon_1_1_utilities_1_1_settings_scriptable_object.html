◆ Single

T Photon.Utilities.SettingsScriptableObject< T >.Single
staticget

Using Single rather than single fires off an check to make sure the singleton SO has been found and is mapped it 'single'.

It also fires off an Initialize() to ensure everything is in order. Do not use Single in a hot path for this reason, but rather use the backing single field instead.
